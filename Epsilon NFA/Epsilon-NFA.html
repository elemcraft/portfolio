Regular Expression Parser

Features

  • Implement an epsilon non-deterministic finite state automaton (ε-NFA) as the core component of the regular expression parser.
  • Utilize depth first search algorithm to retrieve all states within the epsilon closure of the current state and construct the transition table for the input regular expression.
  • Incorporate stack-based checking to verify balanced brackets within the input regular expression.

Environment

  • Java Version: JDK 11
  • External Libraries: Test code would be run using JUnit 4

Outline

  1. Read in the regular expression
  2. Check the validity of the input regular expression and throw an exception if invalid
  3. Transform the regular expression from infix to postfix
  4. Build the ε-NFA
  5. Put the testing string into the ε-NFA built from the regex

NFA Building Blocks

1. Epsilon Block

2. Symbol Block

3. Union Block

4. Concatenation Block

5. Repetition Block (Kleene Star)

6. One Or More Block (Kleene Plus)

Input

A regular expression can consist of:

  • Symbols: lowercase and uppercase letters, numbers, spaces
  • Operators: the alternation operator |, the Kleene star * and Kleene plus operators +, as well as brackets ( and )

Example

Regular Expression = \((a^+|b)(a^*|c)\)

Verbose Mode Example

Regular Expression = \((a^+|b)(a^*|c)\)
